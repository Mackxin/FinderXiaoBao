# 访达小宝

一个极简的 macOS 菜单栏小工具：**在访达（Finder）窗口的空白处操作鼠标，即可返回上一级目录**。

- 仅此一个功能，常驻菜单栏（不占用 Dock）。
- 触发方式：**在访达窗口空白处双击 → 返回上一级**（简单直接，无需切换）。
- 不影响访达任何原有操作：双击文件/文件夹照常打开，只有「空白处双击」会被接管。
- 支持 Apple 芯片与 Intel Mac（通用二进制）。

---

## 使用方法

1. 打开 `访达小宝.app`（双击即可）。
   - 首次打开会弹出「辅助功能」授权提示，请在 **系统设置 → 隐私与安全性 → 辅助功能** 中勾选 `访达小宝`，然后**重启本应用**使其生效。
   - 首次触发返回上一级时，系统会再弹出「自动化 → 访达」授权提示，请点击**允许**。
   - 首次启动会请求「通知」权限（用于操作成功提示），可按需允许。
2. 打开任意访达窗口，把鼠标移到窗口内的**空白处**（不是某个文件/文件夹图标上），**双击**即可返回上一级。
3. 菜单栏会出现一个 ↑ 圆形图标，可在此：
   - **启用 / 停用**功能；
   - **仅图标视图生效**（勾选后，只有在图标视图下的空白处双击才返回；其他视图不触发）；
   - **登录时启动**（勾选后开机自动运行）；
   - 一键打开日志文件夹、各项系统设置；
   - 退出。

> 提示：除菜单里的「登录时启动」外，也可把 `访达小宝.app` 放到「系统设置 → 通用 → 登录项」里。

## 工作原理（简述）

1. 用 `CGEventTap` 全局监听鼠标左键按下，并按系统双击间隔判定双击。
2. 判定为一次有效双击时，用 Accessibility API 取出光标处界面元素：
   - 是否落在**访达**进程上；
   - 是否位于**窗口内的空白背景**（而非文件/文件夹图标）。
3. 满足条件时，通过 AppleScript 让访达返回上一级（`set target of front window to container of target`），并**保持原视图模式**，成功后以通知提示进入的文件夹名。

核心源码见 `Sources/main.swift`。

## 从源码构建（开发者 / 想自己编译的用户）

**环境要求：**
- macOS 13.0 及以上（Ventura / Sonoma / Sequoia / Tahoe）
- 已安装 Xcode 命令行工具（终端执行 `xcode-select --install` 即可，无需完整 Xcode）

**步骤：**
```bash
git clone <你的仓库地址>
cd FinderGoUp          # 即克隆后的项目根目录
bash build.sh
```
脚本会用 `swiftc` 编译出 `访达小宝.app`（Apple 芯片 + Intel 通用二进制，已 **ad-hoc 签名**）。

**关于签名与权限：**
- 本项目使用 **ad-hoc 签名**（无需任何开发者证书），你在本机编译出的 app 可直接运行。
- 首次运行需按「使用方法」一节授予**辅助功能**与**自动化·访达**两项系统权限，且每次重新编译后需重新授权（详见下方常见问题）。
- 想让别人「双击即开、无 Gatekeeper 警告」，需要用 Developer ID 证书签名并公证（notarize），但这不在本仓库默认流程内。

## 常见问题

**Q：空白处双击没反应？**
- 确认已在「辅助功能」中勾选本应用，并**重启应用**使事件监听生效。
- 确认操作的是「空白处」而非图标；点击列视图右侧预览区等也可能被识别为图标而不触发。
- 菜单栏图标 → 「已启用」状态应为启用。

**Q：提示无法创建事件监听？**
- 多半是「辅助功能」权限未授予。打开菜单 → 「打开辅助功能设置」勾选后重启。

**Q：返回后视图模式变了（比如图标变成了列表）？**
- 不会出现。本工具在返回后会强制保持原来的视图模式（图标/列表/分栏/画廊不变）。

**Q：为什么每次重新编译后都要重新授权辅助功能？**
- 本工具使用 ad-hoc 签名（无开发者证书），**每次编译都会改变签名身份**，系统会把它视为「另一个 App」，之前授予的权限随之失效。解决办法：在「辅助功能」设置里先移除 访达小宝，重新打开新版本并勾选，再完全退出重启即可。功能验证通过后建议不要频繁重新编译。

**Q：会收集我的数据吗？**
- 不会。本工具完全本地运行，不联网、不上传任何信息。
