# 访达小宝 v1.1.1

一个极简的 macOS 菜单栏小工具：在访达（Finder）窗口的**空白处**双击鼠标，即可返回上一级目录。

## ✨ 功能特性

- **唯一功能，专注做一件事**：访达空白处双击 → 返回上一级
- **不干扰原有操作**：双击文件 / 文件夹照常打开，只有「空白处双击」会被接管
- **保持视图模式**：返回后图标 / 列表 / 分栏 / 画廊视图不变
- **触发方式**：访达窗口空白处**双击** → 返回上一级（简单直接，无需切换）
- **仅图标视图生效**（可选）：开启后只在图标视图下的空白处双击才返回
- **登录时启动**（可选）：菜单勾选即可开机自启
- **通用二进制**：同时支持 Apple 芯片与 Intel Mac
- **自定义图标**：App 与菜单栏均使用专属图标（红色方块），不再使用系统默认占位图标
- **授权提示优化**：仅在首次未授权时弹窗引导，之后再次打开不再重复打扰

## 📦 安装与运行

**方式一：下载源码自行编译（推荐，无需任何证书）**

```bash
git clone https://github.com/Mackxin/FinderXiaoBao.git
cd FinderXiaoBao
bash build.sh
```

编译产物 `访达小宝.app` 已 ad-hoc 签名，直接双击运行。

**方式二：下载预编译版本（Release 附件）**

1. 在本仓库 **Releases** 页面下载 `访达小宝.app.zip`；
2. 解压得到 `访达小宝.app`；
3. 首次打开请 **右键 → 打开**（ad-hoc 签名会被 Gatekeeper 拦截，需手动确认一次），或在终端执行：
   ```bash
   xattr -cr 访达小宝.app
   ```
   再双击打开。

> 因本工具使用 ad-hoc 签名（无付费开发者证书），从网络下载的 `.app` 会被 Gatekeeper 标记为「来路不明」，上述右键打开或清除隔离属性是必需的一次性操作。

## 🔐 首次运行需授予两项系统权限

1. **辅助功能**：系统设置 → 隐私与安全性 → 辅助功能，勾选「访达小宝」，然后 **完全退出并重启应用** 使其生效（全局鼠标监听必须此项）。
2. **自动化 → 访达**：首次触发返回时弹窗，点击「允许」。
3. 可选：**通知** 权限，用于操作成功的桌面提示。

> ⚠️ 由于使用 ad-hoc 签名，**每次重新编译都会改变签名身份**，之前授予的「辅助功能」权限会随之失效，需先在设置里移除、重新勾选后再重启。

## 💻 系统要求

- macOS 13.0 及以上（Ventura / Sonoma / Sequoia / Tahoe）
- 已安装 Xcode 命令行工具（`xcode-select --install`，无需完整 Xcode）
