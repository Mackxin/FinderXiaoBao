# 访达小宝 v1.1.0

一个极简的 macOS 菜单栏小工具：在访达（Finder）窗口的**空白处**操作鼠标，即可返回上一级目录。

## ✨ 功能特性

- **唯一功能，专注做一件事**：访达空白处双击 / 单击 / 长按 → 返回上一级
- **不干扰原有操作**：双击文件 / 文件夹照常打开，只有「空白处操作」会被接管
- **保持视图模式**：返回后图标 / 列表 / 分栏 / 画廊视图不变
- **触发方式可切换**：双击（默认）/ 单击 / 长按，菜单内即时切换
- **仅图标视图生效**（可选）：开启后只在图标视图下的空白处操作才返回
- **登录时启动**（可选）：菜单勾选即可开机自启
- **通用二进制**：同时支持 Apple 芯片与 Intel Mac

## 📦 安装与运行

**方式一：下载源码自行编译（推荐，无需任何证书）**

```bash
git clone https://github.com/Mackxin/FinderXiaoBao.git
cd FinderXiaoBao
bash build.sh
```

编译产物 `访达小宝.app` 已 ad-hoc 签名，直接双击运行。

**方式二：直接使用已编译的 .app**

> 因本工具使用 ad-hoc 签名（无付费开发者证书），从网络下载的 `.app` 会被 Gatekeeper 标记为「来路不明」。首次打开请 **右键 → 打开**，或在终端执行 `xattr -cr 访达小宝.app` 清除隔离属性后再双击。

## 🔐 首次运行需授予两项系统权限

1. **辅助功能**：系统设置 → 隐私与安全性 → 辅助功能，勾选「访达小宝」，然后 **完全退出并重启应用** 使其生效（全局鼠标监听必须此项）。
2. **自动化 → 访达**：首次触发返回时弹窗，点击「允许」。
3. 可选：**通知** 权限，用于操作成功的桌面提示。

> ⚠️ 由于使用 ad-hoc 签名，**每次重新编译都会改变签名身份**，之前授予的「辅助功能」权限会随之失效，需先在设置里移除、重新勾选后再重启。

## 💻 系统要求

- macOS 13.0 及以上（Ventura / Sonoma / Sequoia / Tahoe）
- 已安装 Xcode 命令行工具（`xcode-select --install`，无需完整 Xcode）
